# 🤖 Echo Bot - Шаблон Telegram бота

Минимальный эхо-бот, построенный по модульной архитектуре. Используйте как основу для новых проектов.

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
cd echo_bot/bot
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

```bash
# Скопируйте пример и заполните
cp example.env .env
```

Отредактируйте `.env`:
```
BOT_TOKEN=ваш_токен_от_BotFather
```

### 3. Запуск бота

```bash
python app.py
```

## 🏗️ Архитектура

### Ключевая концепция: **Виджет = Оркестратор компонентов**

Виджет в `handler/` не содержит логику напрямую, а собирает компоненты из `node/`:

```
┌─────────────────────────────────────────────┐
│        handler/echo_widget.py               │
│              (Оркестратор)                  │
├─────────────────────────────────────────────┤
│  1. Импортирует компоненты из node/         │
│  2. Создает реестр Answer-ов                │
│  3. Вызывает: Trigger → Code → Answer       │
│  4. Регистрируется в роутере                │
└─────────────────────────────────────────────┘
         ↓              ↓              ↓
    ┌─────────┐   ┌─────────┐   ┌─────────┐
    │ Trigger │   │  Code   │   │ Answer  │
    │ (node/) │   │ (node/) │   │ (node/) │
    └─────────┘   └─────────┘   └─────────┘
```

### Компоненты node/

| Компонент | Ответственность |
|-----------|-----------------|
| **Trigger** | Визуальные операции: сброс FSM, извлечение данных |
| **Code** | Логика: вызов API-клиентов, роутинг к Answer |
| **Answer** | Отрисовка UI: текст, клавиатуры, медиа |

### Система тегов

Компоненты группируются по функциональным областям (тегам):
- `control` - управляющие команды (/start, /help, эхо)
- `payment` - платежи (пример для расширения)
- `vacancy` - вакансии (пример для расширения)

## 📝 Функционал бота

| Команда/Действие | Описание |
|------------------|----------|
| `/start` | Приветственное сообщение |
| `/help` | Справка по боту |
| Любой текст | Эхо-ответ |

## 🔧 Расширение

### Добавление нового функционала

1. **Определить тег** (например, `payment`)

2. **Создать компоненты в node/**:
   ```
   node/payment/trigger/process_payment_trigger.py
   node/payment/code/process_payment_code.py
   node/payment/answer/payment_success_answer.py
   ```

3. **Создать виджет в handler/**:
   ```
   handler/v1/user/payment/process_payment_widget.py
   ```

4. **Создать состояния** (если нужны):
   ```
   state/payment.py
   ```

5. **Создать колбеки** (если нужны):
   ```
   callback/payment.py
   ```

6. **Подключить роутер** в `handler/include_router.py`

### Добавление API-клиента (если фича работает с бэкендом)

Бот не подключается к БД напрямую. Для работы с данными создаётся бэкенд-сервис, а в боте — API-клиент:

```
service/notes_api/
├── __init__.py
├── notes_client.py      # HTTP-методы к бэкенду
└── schemas.py           # Pydantic-схемы запросов/ответов
```

Подробнее: [docs/service.md](docs/service.md)
